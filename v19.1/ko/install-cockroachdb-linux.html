마지막 릴리즈, {{ page.release_info.version }} 에 추가된 내용을 알고 싶으면 릴리즈 노트를 보십시오. 오래된 버전에서 이 릴리즈로 업그레이드하려면 클러스터 업그레이드를 보십시오.

실행파일 다운로드

{% include {{ page.version.version }}/misc/linux-binary-prereqs.md %}

  1. 리눅스 칵로치디비 아카이브를 다운로드받고, 압축을 푸십시오:

    copy
    $ wget -qO- https://binaries.cockroachdb.com/cockroach-{{page.release_info.version}}.linux-amd64.tgz | tar  xvz

  2. 실행파일을 당신의 PATH에 복사하여 칵로치 명령어를 아무 쉘에서나 사용할 수 있게 하십시오:

    {% include copy-clipboard.html %}
    cp -i cockroach-{{ page.release_info.version }}.linux-amd64/cockroach /usr/local/bin

    권한 에러가 나오면 sudo를 이용해 명령어를 사용하십시오.

  3. 칵로치 디비 릴리즈와 모범사례를 최신으로 유지하십시오:

쿠버네티스 사용하기

설정 파일 또는 헬름 패키지 매니저를 통해 쿠버네티스로 칵로치디비를 오케스트레이션하려면, 아래 튜토리얼을 사용하십시오:

  • 미니쿠베를 이용한 칵로치디비 로컬 오케스트레이션
  • 단일 쿠버네티스 클러스트를 이용한 칵로치디비 오케스트레이션
  • 멀티 쿠버네티스 클러스터를 가로지르는 칵로치디비 오케스트레이션

도커 사용하기

{{site.data.alerts.callout_danger}} 칵로치디비와 같은 스테이트풀 애플리케이션을 도커에서 사용하는 것은 일반적인 도커 사용보다 더 복잡하고 오류가 발생하기 쉽습니다. 도커에 매우 숙련되어 있지 않다면, 다른 설치 및 배포 방법으로 시작하시는 것을 권고드립니다.{{site.data.alerts.end}}

  1. 리눅스에 도커를 설치한다. 모든 전제조건을 충족했는지 확인해 주십시오.

  2. 도커 데몬이 백그라운드에서 동작하는지 확인하십시오:

    copy
    $ docker version

    서버 목록이 보이지 않으면, Docker 데몬을 시작하십시오.

    {{site.data.alerts.callout_info}}리눅스에서, 도커는 sudo 권한을 요구합니다.{{site.data.alerts.end}}

  3. 칵로치디비 {{page.release_info.version}} 릴리즈를 도커 허브에서 다운로드받으십시오:

    copy
    $ sudo docker pull {{page.release_info.docker_image}}:{{page.release_info.version}}

  4. 칵로치 디비 릴리즈와 모범사례를 최신으로 유지하십시오:

소스에서 빌드하기

  1. 다음 요구사항을 설치하십시오:

    C++ 컴파일러 C++ 11을 지원해야 합니다. 6.0 이전의 GCC는 이 이슈 때문에 동작하지 않습니다. 맥OS에서는 Xcode로 충분합니다.
    Go 버전 1.11.6 이상이 필요합니다.
    Bash 버전 4 이상이 선호되지만, 3.x 이후 릴리즈도 동작하는 것으로 알려져 있습니다.
    CMake 버전 3.81 이상이 동작하는 것으로 알려져 있습니다.
    Autoconf 버전 2.68 이상이 필요합니다.

    64비트 시스템을 강력히 권장합니다. 32비트 시스템에서 칵로치디비를 빌드 또는 실행하는 것은 테스트되지 않았습니다. 또 2GB 이상의 RAM이 필요합니다. 테스트를 실행하려면 4GB에 가까운 RAM이 필요합니다.

  2. 칵로치디비 {{ page.release_info.version }} 소스 아카이브를 다운로드 받고, 압축을 푸십시오:

    copy
    $ wget -qO- https://binaries.cockroachdb.com/cockroach-{{page.release_info.version}}.src.tgz | tar  xvz

  3. 압축이 풀린 디렉토리에서, make build를 실행하십시오:

    {% include copy-clipboard.html %}
    cd cockroach-{{ page.release_info.version }}
    {% include copy-clipboard.html %}
    make build

    빌드 프로세스에 10분 이상이 소요되니, 잠시 기다려 주십시오.

    {{site.data.alerts.callout_info}}기본 실행파일에는 아파치 라이센스 2(APL2)인 코어 오픈소스 기능과 칵로치디비 커뮤니티 라이센스(CCL) 라이센스인 엔터프라이즈 기능이 포함되어 있습니다. 순수 오픈소스(APL2) 버전을 엔터프라이즈 기능없이 빌드하려면 make buildoss를 사용하십시오. 이 블로그 포스트에서 세부내용을 확인할 수 있습니다.{{site.data.alerts.end}}

  4. cockroach 실행파일을 /usr/local/bin에 설치해 칵로치 명령어를 어떤 디렉토리에서든 쉾게 실행할 수 있게 하십시오:

    {% include copy-clipboard.html %}
    make install

    권한 에러가 발생한다면 sudo 접두어를 사용하십시오.

    당신은 빌드된 디렉토리에서 cockroach 실행파일을 실행할 수도 있지만, ./src/github.com/cockroachdb/cockroach/cockroach, 이어지는 문서는 실행파일이 PATH에 있다고 가정합니다.

  5. 칵로치 디비 릴리즈와 모범사례를 최신으로 유지하십시오:

무엇을 더 알아볼까요?
